---
title: "Listar Pagamentos"
api: "GET /api/v1/payment/list"
description: "Retorna a lista de todos os pagamentos do usuário autenticado."
---

Este endpoint retorna todos os pagamentos associados à conta autenticada.

Ele permite:

- listar transações
- filtrar por status
- paginar resultados
- visualizar estatísticas gerais de pagamentos

Os resultados são retornados **do mais recente para o mais antigo**.

---

# Autenticação

Este endpoint exige autenticação obrigatória.

Envie sua chave de API no header da requisição:

```bash
X-API-Key: zynx_sua_chave_aqui
```

Se nenhuma chave válida for enviada:

```json
{
  "error": "Unauthorized"
}
```

---

# Rate Limit

Para evitar abuso da API existe limite de requisições.

| Tipo | Limite |
|-----|------|
API Key | 100 requisições por minuto |

Se o limite for excedido:

```json
{
  "error": "Too many requests"
}
```

---

# Parâmetros de Query

<ParamField query="status" type="string">
Filtra pagamentos por status.

Valores aceitos:

```
PENDING
ACTIVE
COMPLETED
PAID
```

Exemplo:

```
?status=COMPLETED
```
</ParamField>

<ParamField query="limit" type="number" default="50">
Define quantos pagamentos serão retornados por requisição.

Exemplo:

```
?limit=10
```
</ParamField>

<ParamField query="offset" type="number" default="0">
Define o ponto inicial da paginação.

Exemplo:

```
?offset=20
```

Isso permite carregar páginas de resultados.
</ParamField>

---

# Ordenação

Os pagamentos são ordenados automaticamente por:

```
createdAt DESC
```

Ou seja, **os pagamentos mais recentes aparecem primeiro**.

# Estrutura de Pagamento

Cada objeto dentro de `payments` possui os seguintes campos.

<ResponseField name="id" type="string">
ID único do pagamento.
</ResponseField>

<ResponseField name="correlationID" type="string">
Identificador da transação utilizado internamente.
</ResponseField>

<ResponseField name="value" type="number">
Valor total da transação em centavos.
</ResponseField>

<ResponseField name="netValue" type="number">
Valor líquido da transação.
</ResponseField>

<ResponseField name="fee" type="number">
Taxa aplicada na transação.
</ResponseField>

<ResponseField name="status" type="string">
Status atual do pagamento.
</ResponseField>

<ResponseField name="description" type="string">
Descrição do pagamento.
</ResponseField>

<ResponseField name="createdAt" type="string">
Data de criação da transação.
</ResponseField>

<ResponseField name="updatedAt" type="string">
Data da última atualização da transação.
</ResponseField>

---

# Estrutura de Paginação

<ResponseField name="total" type="number">
Quantidade total de pagamentos encontrados.
</ResponseField>

<ResponseField name="limit" type="number">
Quantidade máxima retornada por requisição.
</ResponseField>

<ResponseField name="offset" type="number">
Posição inicial da consulta.
</ResponseField>

<ResponseField name="hasMore" type="boolean">
Indica se existem mais resultados disponíveis.
</ResponseField>

---

# Estatísticas

A resposta também inclui estatísticas agregadas.

<ResponseField name="total" type="number">
Quantidade total de pagamentos.
</ResponseField>

<ResponseField name="completed" type="number">
Quantidade de pagamentos concluídos.
</ResponseField>

<ResponseField name="pending" type="number">
Quantidade de pagamentos pendentes.
</ResponseField>

<ResponseField name="totalValue" type="number">
Soma de todos os valores de pagamentos.
</ResponseField>

<ResponseField name="totalNetValue" type="number">
Soma de todos os valores líquidos recebidos.
</ResponseField>

# Boas práticas

- utilize paginação para evitar respostas muito grandes  
- filtre por status quando necessário  
- utilize o campo `hasMore` para implementar scroll infinito ou carregamento progressivo  
- armazene os IDs das transações para consultas individuais futuras

---

Este endpoint permite consultar **histórico completo de pagamentos** de forma segura e organizada dentro da plataforma.
