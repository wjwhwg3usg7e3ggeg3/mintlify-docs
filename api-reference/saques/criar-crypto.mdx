---
title: "Criar Saque Crypto"
api: "POST /api/v1/withdraw/crypto"
description: "Solicita um saque em USDT pela rede BEP20."
---

Este endpoint permite sacar saldo da conta para uma **carteira USDT na rede BEP20**.

Quando o saque é solicitado, a API:

1. valida o valor
2. valida o endereço da wallet
3. verifica se existe saque em processamento
4. calcula a taxa
5. debita o saldo da conta
6. envia o saque para a gateway
7. adiciona o saque na fila de processamento

---

# Autenticação

Envie sua API Key no header.

```bash
X-API-Key: zynx_sua_chave_aqui
```

---

# Limites de saque

| Regra | Valor |
|------|------|
Valor mínimo | R$20 |
Valor máximo | R$3000 |

---

# Taxa

A taxa aplicada no saque crypto é:

```
6% + R$2,00
```

Exemplo:

| Saque | Taxa | Total debitado |
|------|------|------|
R$100 | R$8 | R$108 |

---

# Parâmetros

<ParamField body="amount" type="number" required>
Valor do saque em reais.
</ParamField>

<ParamField body="wallet" type="string" required>
Endereço da carteira USDT na rede BEP20.
</ParamField>

<ParamField body="description" type="string">
Descrição opcional do saque.
</ParamField>

---

# Formato da wallet

A carteira deve seguir o padrão:

```
0x + 40 caracteres hexadecimais
```

Regex usada pela API:

```
^0x[a-fA-F0-9]{40}$
```

Exemplo válido:

```
0x742d35Cc6634C0532925a3b844Bc454e4438f44e
```

# Exemplo de requisição

<RequestExample>

```bash
curl -X POST "https://api.zynxpay.lat/api/v1/withdraw/crypto" \
-H "Content-Type: application/json" \
-H "X-API-Key: zynx_sua_chave_aqui" \
-d '{
"amount": 100,
"wallet": "0x742d35Cc6634C0532925a3b844Bc454e4438f44e",
"description": "Saque USDT"
}'
```

</RequestExample>

# Status possíveis

| Status | Descrição |
|------|-----------|
QUEUED | saque adicionado à fila |
PROCESSING | saque sendo processado |
COMPLETED | saque concluído |
FAILED | saque falhou |

# Erros possíveis

```json
{
  "error": "Wallet inválida",
  "message": "O endereço da wallet deve estar no formato BEP20 (0x seguido de 40 caracteres hexadecimais)"
}
```
