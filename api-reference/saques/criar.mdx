---
title: "Criar Saque PIX"
api: "POST /api/v1/withdraw"
description: "Solicita um saque do saldo da conta para uma chave PIX."
---

Este endpoint permite retirar saldo da conta para uma **chave PIX**.

Quando um saque é solicitado, o sistema:

1. valida o valor
2. verifica se existe outro saque em processamento
3. debita o saldo da conta
4. envia a solicitação para a gateway de pagamento
5. atualiza o status do saque

---

# Autenticação

Envie sua API Key no header da requisição.

```bash
X-API-Key: zynx_sua_chave_aqui
```

---

# Limite de saque

| Regra | Valor |
|------|------|
Valor mínimo | R$1,00 |

---

# Parâmetros

<ParamField body="amount" type="number" required>
Valor do saque em reais.

Exemplo:

```
10
25.50
100
```
</ParamField>

<ParamField body="pixKey" type="string" required>
Chave PIX de destino.
</ParamField>

<ParamField body="pixKeyType" type="string" required>
Tipo da chave PIX.

Valores possíveis:

```
CPF
EMAIL
PHONE
RANDOM
COPYPASTE
```
</ParamField>

<ParamField body="description" type="string">
Descrição opcional do saque.

Máximo de **100 caracteres**.
</ParamField>

<ParamField body="verificationCode" type="string">
Código de verificação enviado por e-mail (caso segurança esteja ativada).
</ParamField>

---

# Exemplo de requisição

<RequestExample>

```bash
curl -X POST "https://api.zynxwallet.com.br/api/v1/withdraw" \
-H "Content-Type: application/json" \
-H "X-API-Key: zynx_sua_chave_aqui" \
-d '{
"amount": 50,
"type": "pix",
"pixKey": "email@exemplo.com",
"pixKeyType": "EMAIL",
"description": "Saque"
}'
```

</RequestExample>

---

# Exemplo de resposta

<ResponseExample>

```json
{
"success": true,
"message": "Saque solicitado com sucesso e está sendo processado.",
"withdrawId": "wd_abc123",
"status": "PROCESSING"
}
```

</ResponseExample>

---

# Status possíveis

| Status | Descrição |
|------|-----------|
PENDING | saque criado |
PROCESSING | sendo processado |
FAILED | falha no saque |
