---
title: "Enviar Pagamento"
api: "POST /api/v1/payment/send"
description: "Envia dinheiro via PIX a partir do saldo da conta."
---

Este endpoint permite **enviar dinheiro ou realizar reembolsos via PIX** usando o saldo da conta.

Fluxo do envio:

1. valida os campos obrigatórios
2. calcula a taxa da transação
3. debita o saldo do usuário
4. envia o pagamento pela gateway
5. registra auditoria
6. dispara webhook

---

# Autenticação

Este endpoint funciona **apenas com API Key**.

Header obrigatório:

```bash
X-API-Key: zynx_sua_chave_aqui
```

---

# Rate Limit

| Tipo | Limite |
|-----|------|
API Key | 30 requisições por minuto |

---

# Parâmetros

<ParamField body="amount" type="number" required>
Valor do envio em reais.
</ParamField>

<ParamField body="pixKey" type="string" required>
Chave PIX de destino.
</ParamField>

<ParamField body="pixKeyType" type="string" required>
Tipo da chave PIX.

Valores aceitos:

```
CPF
EMAIL
PHONE
RANDOM
CNPJ
```
</ParamField>

<ParamField body="description" type="string">
Descrição opcional da transação.

Máximo de **100 caracteres**.
</ParamField>

---

# Exemplo de requisição

<RequestExample>

```bash
curl -X POST "https://api.zynxpay.lat/api/v1/payment/send" \
-H "Content-Type: application/json" \
-H "X-API-Key: zynx_sua_chave_aqui" \
-d '{
"amount": 100,
"pixKey": "email@exemplo.com",
"pixKeyType": "EMAIL",
"description": "Reembolso"
}'
```

</RequestExample>

---

# Como a taxa funciona

A taxa é calculada automaticamente com base no **plano do usuário**.

Fluxo:

```
valor enviado = valor solicitado - taxa
```

Exemplo:

| Campo | Valor |
|------|------|
valor solicitado | R$100 |
taxa | R$2 |
valor enviado | R$98 |

# Estrutura da resposta

<ResponseField name="data.amount" type="number">
Valor total debitado da conta em centavos.
</ResponseField>

<ResponseField name="data.fee" type="number">
Taxa cobrada pela transação.
</ResponseField>

<ResponseField name="data.sent" type="number">
Valor efetivamente enviado após desconto da taxa.
</ResponseField>

<ResponseField name="data.destination" type="string">
Chave PIX de destino.
</ResponseField>

<ResponseField name="data.mistic" type="object">
Resposta retornada pela gateway de pagamento.
</ResponseField>

# Erros possíveis

## Valor ausente

```json
{
  "error": "VALOR_AUSENTE",
  "message": "O campo \"amount\" é obrigatório"
}
```

---

## PIX ausente

```json
{
  "error": "PIX_AUSENTE",
  "message": "Os campos \"pixKey\" e \"pixKeyType\" são obrigatórios"
}
```

---

## Valor inválido

```json
{
  "error": "Valor inválido",
  "message": "O valor deve ser maior que zero"
}
```

---

## Saldo insuficiente

```json
{
  "error": "SALDO_INSUFICIENTE",
  "message": "Você não possui saldo suficiente para realizar este envio."
}
```

---

## Erro na gateway

```json
{
  "error": "Erro ao enviar via Mistic",
  "message": "Falha na transferência"
}
```

---

Este endpoint é usado para **enviar pagamentos ou realizar reembolsos diretamente via PIX utilizando o saldo da conta**.
